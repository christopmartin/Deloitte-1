Data Flow — Document → Build Spec

The core ingestion pipeline. Each step is explicitly an AI Agent or deterministic code; humans gate the transition into the real design tables.

AI Agent (Claude) Deterministic code
01

Upload & read document

Deterministic

User uploads a .docx/text file via multer; document-reader (mammoth) extracts plain text into asdlc_ingest_document.

02

Extract design entities

AI Agent

The Extraction AI Agent (claude-processor) uses Claude tool-use to emit structured entities — use cases, workflows, agents, tools, requirements — staged in asdlc_ingest_extraction. Tools & field maps come from the deterministic entity-registry.

03

Two-pass reconciliation

AI Agent

For every update/delete against existing design, the server loads the full current record and the Reconciler AI Agent re-emits a complete, conflict-free version — no stale fields left behind.

04

Clarification Q&A rounds

AI Agent

Where confidence is low, the agent raises clarification questions (asdlc_ingest_clarification); human answers feed a multi-round AI re-process until the extraction settles.

05

Promote to a Change Packet

Deterministic

Before promoting, agent/quality-check.js runs a deterministic gate over the staged batch — the same rule set as the AI's ad-hoc quality review, made repeatable: same real-world entity extracted twice under different names (e.g. a workflow split across two rounds with children divided between them), duplicate tools/integrations, unresolved "TBD" placeholders, AI-invented agents/workflows/tools not named in the source, and functional requirements with no acceptance criterion or test case. Structural ("block") findings can never be bypassed; advisory ("warn") findings require one explicit acknowledgment (acknowledge_warnings:true) before promotion proceeds. This is separate from and runs before the post-promotion Quality Reviewer AI Agent (step 06) — deterministic and free vs. an LLM pass over the finished Change Packet.

Promotion itself resolves target slugs and bundles all staged entities into one Change Packet (pending_review) with per-item create / update / delete diff items.

06

Quality review

AI Agent

The Quality Reviewer AI Agent checks requirements, workflow steps and derived elements for conflicts, gaps and validity, attaching findings for the human reviewer.

07

Human approves → materialize

Deterministic

On approval, applyChangePacket() writes into the real design tables (insert / partial update / soft-delete), bumps the project version and records a full field-level audit log.

08

Auto-generate tests & links

AI Agent

Fire-and-forget after commit: the Test-Generator and Traceability AI Agents synthesize test cases and requirement↔element links for newly materialized entities.

09

Baseline & Build Export

Deterministic

A reviewed version is locked as a baseline; Build Export emits a Markdown build spec for Claude Code / ServiceNow, with an optional AI design-review appendix.

05

AI Agents vs. Deterministic Code

A clear seam runs through the system: Claude reasons and proposes; deterministic code resolves, gates and persists.

Ingestion & Design AI Agents

  • Extraction — entities from documents · Sonnet 4.6 (default)
  • Reconciler — full re-emit on updates
  • Quality Reviewer — conflicts & gaps
  • Test Generator — coverage synthesis
  • Req-Linker — traceability · Haiku 4.5
  • RASIC & Swimlane derivers — role/lane inference

ServiceNow Round-Trip AI Agents

  • Discovery Planner — ONE Ingest Document's own pre-promote requirements + real inventory → a focused import slice, incl. related/supporting tables via a reference-relationship graph; genuine ambiguity raises a real discovery: clarification in that document's own Q&A (never the project-wide requirements, never a passive note) · Opus 4.8 + thinking (human reviews/approves before any capture; assessment runs automatically if none covers the scope; #106/#108)
  • Reverse-Engineer — live SN app → design · Opus 4.8 + thinking (interpretive types only: AI agents, use cases, tools; #103)
  • Reconcile — merge against Workbench design · Opus 4.8
  • Review — validate reconciled output · Opus 4.8
  • Explain with AI — on-request narration of one business-logic rule · Opus 4.8 (#103)

Preview/Run now show a pre-flight time/cost estimate + confirm step, a live progress meter, and a cancel button — a preview is not free, it runs the same AI stages as a real run and only skips the DB write (#105).

Deterministic (non-AI) Code

  • entity-registry — single source of truth
  • /promote — slug resolution & diffing
  • applyChangePacket() — materializer
  • audit log — field-level history
  • capture · gate · seed — pipeline plumbing
  • sys_metadata sweep — completeness + per-record change signals (#86)
  • sn-direct-map — deterministic field-copy for literal SN types (catalog items, tables, forms, integrations, business logic); zero-AI capture, narrative on request (#103)
  • readReferenceGraph — reads this instance's real `sys_dictionary` reference fields, no AI (#106)
