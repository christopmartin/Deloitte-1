## High

1. **[correctness] [claude-processor.js:985](../../backend-node/agent/claude-processor.js)** — Pass-2 reconciliation does `deleteExtraction(oldId)` then `upsertExtraction(...)` per item with no transaction; if the upsert throws mid-list, the pass-1 staged row is already gone, and the outer catch's claim "keeping pass-1 updates" (line 991) is false for that item — the staged extraction is silently lost. → Update the existing row in place (or wrap each replace in BEGIN/COMMIT) instead of delete-then-insert.

2. **[correctness] [processor.js:15](../../backend-node/agent/processor.js)** — API-key validation is inconsistent across modules: `processor.js` checks only the `'paste-your-anthropic-key-here'` placeholder, `claude-processor.js:31` checks only presence, while `quality-reviewer.js:42`, `cross-check.js:43`, `rasic-deriver.js:45`, etc. also reject `'your_anthropic_api_key_here'` and whitespace. A key set to the second placeholder routes ingestion to the real engine, which then fails the document instead of falling back to stub. → Extract one shared `hasKey()` and use it everywhere (7+ copies today).
