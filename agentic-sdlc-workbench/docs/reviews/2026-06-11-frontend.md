**Security verdict — clean.** A shared `escHtml` helper ([app.js:443](agentic-sdlc-workbench/frontend/app.js)) exists and is used on every data-bearing error string; all dynamic API/extraction data is otherwise rendered through the `el()` builder ([app.js:344](agentic-sdlc-workbench/frontend/app.js)) which uses `textContent`/`createTextNode`. Across ~250 `innerHTML`/`insertAdjacentHTML` sites, **no unescaped rendering of ingested-document or AI-generated text was found** — the XSS surface is well-contained. All network I/O routes through the shared `apiFetch` wrapper ([app.js:243](agentic-sdlc-workbench/frontend/app.js)) which checks `res.ok` and tolerates non-JSON bodies, so the backend's missing error middleware doesn't leave the UI parsing empty 500s.

## High

1. **[correctness] [baseline.js:208](agentic-sdlc-workbench/frontend/modules/baseline.js)** — `loadBaselineDetail` fetches `/baselines/${baseline.id}`, but the list API populates `baseline_id` (used everywhere else: lines 130, 172, 287, 357). `baseline.id` is `undefined`, so the detail fetch hits `/baselines/undefined`, fails, and the `.catch(() => baseline)` silently falls back to the stale list row — the detail/compare panes never show fresh server data. → Change to `${baseline.baseline_id}`.

2. **[correctness] [ingest.js:920](agentic-sdlc-workbench/frontend/modules/ingest.js)** — The clarifications/answer POST discards its response. The backend now returns `standing_applied` (summarizing the project-wide standing-question fills, added in the #40 fix), so the user gets **no feedback** that answering a clarification just changed `runs_per_period`/`cost_model` across the project. → Capture the result and, when `standing_applied?.length`, surface it (toast or summary block) before re-rendering. *(Relates to #40 / #48.)*

## Medium

3. **[correctness] [cost_management.js:135](agentic-sdlc-workbench/frontend/modules/cost_management.js)** — The edit-skill modal uses class names `_p2Overlay`/`_p2Header`/`_p2Close`, but those are **JS helper functions in design_review.js**, not CSS classes (absent from styles.css). The modal renders with no overlay backdrop, no centering, and an unstyled close button — it appears as a raw block at the bottom of the page. → Use the real modal classes (e.g. `modal-overlay`/`modal-card` as in [library.js:194](agentic-sdlc-workbench/frontend/modules/library.js)) or a shared modal helper (see #7). *(Root cause is the lack of a shared modal helper — finding 7.)*

4. **[correctness] [app.js:64](agentic-sdlc-workbench/frontend/app.js)** — `navigate()` catches a module's `render()` throw and shows an error state, but does **not** reset `currentModule`; the failed module stays active, and a project-selector change ([app.js:164](agentic-sdlc-workbench/frontend/app.js)) re-invokes the same throwing module. There's also no global `unhandledrejection` handler, so async throws after `render()` resolves surface only in the console. → Clear `currentModule` on error and add a top-level `unhandledrejection` logger.

5. **[correctness] [testing.js:59](agentic-sdlc-workbench/frontend/modules/testing.js)** — `renderAcView`/`render` treat `/projects` as a bare array (`projects.forEach`), while every other module handles both `[]` and `{items:[]}` (e.g. [ingest.js:201](agentic-sdlc-workbench/frontend/modules/ingest.js), change_packets.js:322). If that endpoint ever returns the paginated shape, the project dropdown silently empties and the tab is unusable. → Normalize: `const projects = Array.isArray(data) ? data : (data.items || [])`.

6. **[structure] [design_review.js](agentic-sdlc-workbench/frontend/modules/design_review.js)** — One 6,151-line module owns the entire Design Review feature: the tab controller, 15 per-entity report renderers, a hand-rolled markdown renderer + doc drawer (2343–2629), 9 modal editors (3340–4406), the exceptions/quality subsystem (496–656, 5856–5981), CP history + detail modals (5986–6145), and ~1,170 lines of CSS in `injectStyles` (4644–5821). → Split into `design_review_styles.js` (the CSS), `dr_markdown.js`, `dr_modals.js`, `dr_quality.js`, `dr_history.js`, and one renderer module per entity-family, leaving `design_review.js` as the controller.

7. **[structure] no shared modal helper** — Overlay scaffolding (create overlay → modal → header → close button → outside-click-to-close → `body.appendChild`) is hand-rolled 10+ times: [design_review.js:3341](agentic-sdlc-workbench/frontend/modules/design_review.js) (+5 more in that file), [change_packets.js:456](agentic-sdlc-workbench/frontend/modules/change_packets.js), [testing.js:693](agentic-sdlc-workbench/frontend/modules/testing.js), with divergent class names — and cost_management.js mis-references design_review's private helpers (finding 3). → Extract one `openModal({title, body, footer})` → `{overlay, body, close}` into a shared `ui.js`.

8. **[structure] duplicated UI/format/unwrap helpers across ~10 modules** — `stat()` copy-pasted ([admin_ai.js:208](agentic-sdlc-workbench/frontend/modules/admin_ai.js), admin_best_practices.js:244); the `meta-grid` field loop repeated in evidence/audit/validation/baseline; token/cost formatters reimplemented in admin_ai/projects/cost_management; and the API-envelope normalization `Array.isArray(data) ? data : (data.items || [])` appears in ~10 modules. → Promote `stat`, `metaGrid(rows)`, currency/token formatters, and a shared `unwrap(data, key)` into app.js alongside `el`/`renderTable`.

9. **[structure] [testing.js:516](agentic-sdlc-workbench/frontend/modules/testing.js)** — Inline-editable-cell logic (blur-to-save, Enter/Escape, revert-on-failure, source-pill refresh) is duplicated between `buildAcRow` (203) and `editableCell` (516), ~3× total. → Extract one `makeEditable(initialText, { onSave })` helper.

10. **[structure] [change_packets.js:751](agentic-sdlc-workbench/frontend/modules/change_packets.js)** — `renderChangeItem` is ~215 lines (header, field tables, diff, conflict box, SN-sync diff, per-item approve/reject confirm), and its inline-confirm blocks (904–955) are near-identical to the CP-level confirm in `renderDetail` (1150–1235). → Split into `buildItemHeader`/`buildItemBody`/`buildItemDecisionFooter` + a shared `inlineConfirm(label, cls, onYes)`.

11. **[structure] [evidence.js:186](agentic-sdlc-workbench/frontend/modules/evidence.js), [reports.js:158](agentic-sdlc-workbench/frontend/modules/reports.js)** — Two action buttons are **stubs that fake success**: evidence's "Re-run Extraction" does `await new Promise(r=>setTimeout(r,800))` then toasts "Extraction queued successfully"; reports' "Preview" toasts "Preview generated (stub)". They tell the user an action succeeded when nothing happened. → Wire to real endpoints or disable/label as not-implemented. *(Evidence is already tracked as backlog #20.)*

12. **[performance] full re-render on single-item edits / filter clicks** — Editing/adding one item re-fetches and rebuilds the whole view: [testing.js:583](agentic-sdlc-workbench/frontend/modules/testing.js) (and 460, 325) re-runs `renderTcView()` on every TC edit; the design-review quality panel rebuilds entirely on each filter-chip click ([design_review.js:5969](agentic-sdlc-workbench/frontend/modules/design_review.js)). → Targeted DOM updates (append/replace the one row; toggle chip state + call the inner `reload()`), as `addAc` already does at testing.js:281.

13. **[performance] [library.js:83](agentic-sdlc-workbench/frontend/modules/library.js)** — Search filters run `JSON.stringify(item).toLowerCase().includes(...)` over **every row on every keystroke** (also [evidence.js:77](agentic-sdlc-workbench/frontend/modules/evidence.js), [validation.js:152](agentic-sdlc-workbench/frontend/modules/validation.js)), then fully re-render — O(rows × payload) per character. → Pre-compute a lowercased search string per row at load and debounce the input handler.

## Low

18. **[performance] redundant repeated DOM queries in hot paths** — [design_review.js:407](agentic-sdlc-workbench/frontend/modules/design_review.js) re-queries the just-created button row for all 15 tabs inside a `forEach`; [testing.js:535](agentic-sdlc-workbench/frontend/modules/testing.js) re-walks `td.closest('tr')` then `querySelector` in every blur handler. → Capture the node in a closure at build time.

19. **[hygiene] inconsistent toast severity strings** — `showToast(…, 'warning')` ([testing.js:454](agentic-sdlc-workbench/frontend/modules/testing.js)) vs `'warn'` ([validation.js:290](agentic-sdlc-workbench/frontend/modules/validation.js), build_export.js:583); only one matches the CSS class `showToast` emits, so the other renders unstyled. → Standardize on the value the CSS defines.

20. **[hygiene] dead code + style-idiom drift** — Unused `const _orig = setDroppedFile` ([ingest.js:371](agentic-sdlc-workbench/frontend/modules/ingest.js)); `el()` style passed as an object in some calls and a raw CSS string in others (change_packets.js:1095 vs 671); `alert()` used for the swimlane error path ([design_review.js:1714](agentic-sdlc-workbench/frontend/modules/design_review.js)) where every other error uses `showToast`. → Remove dead code; pick one style idiom; use `showToast` consistently.

21. **[security] [design_review.js:1187](agentic-sdlc-workbench/frontend/modules/design_review.js)** (and 1358) — A `typeof x === 'string' ? x : JSON.stringify(x)` fallback renders non-string risk/decision entries as a raw `{...}` blob. **Not** an XSS vector (it goes through the safe `el()`/textContent path), but the user sees stringified JSON instead of readable text. → Render a meaningful field (`r.risk`/`r.description`).

22. **[hygiene] [home.js:45](agentic-sdlc-workbench/frontend/modules/home.js)** — The post-apply banner's dismissal state (`getDismissedPostApply`) is stored globally, keyed only by packet code, not per-project — low impact but can hide a banner across projects. → Key the dismissal by project + packet.
