Agentic SDLC Workbench — Main Flow

One end-to-end process, from application setup through design approval, iteration, and deployment back to ServiceNow. Setting up from scratch (greenfield) happens once — after that first deploy to a Dev instance, the application exists in ServiceNow and every new requirement follows the ServiceNow-linked (brownfield) path. Admin flows not shown.

Greenfield once, then brownfield forever. The very first pass for a brand-new app skips the ServiceNow steps (Phase 3 and the deploy-back), because there's nothing to map against yet. Once that first design is deployed to a ServiceNow Dev instance, the app exists in ServiceNow — so every later requirement re-enters at Ingest and takes the linked path: checked against the live instance, mapped to what's already there, and deployed back. The two decision diamonds "App linked to ServiceNow?" are where that switch happens.

The Duplicate & Conflict Check looks two ways. One automatic check applies two lenses at once — is this already covered by our own existing design, and (when linked) is it already built and live in ServiceNow? Either real match blocks promotion, so the same thing is never built twice.

Assessment is automatic; Catalog Scan is a deploy-time safeguard. Generating the import plan quietly assesses the instance for you — there's no separate step. The optional Catalog Scan earns its place on the way out: a whole-instance inventory so the Build Spec can warn if a deploy would create something that already exists on the instance.

The ServiceNow steps are mostly people, not automation. A teal-outlined step that's blue is a deliberate human action — a person generates the import plan, reviews and approves which records get pulled, and launches the sync (after seeing the cost estimate, with cancel available). Only the purple steps ("System Pulls Records," "System Reconciles") run unattended — and even those hand their results to a human review queue before anything changes the design.

✨ marks every step that calls AI: extracting entities from a requirement, the duplicate/conflict check (both lenses), generating the ServiceNow import plan, and pulling/reconciling ServiceNow data (AI is used there only for the interpretive record types — agents, use cases, tools, workflow intent; plain structured records are copied as-is, no AI). Every other step is a person deciding or the system assembling/packaging — no model call involved.
